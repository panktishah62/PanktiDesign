Apex Filters

Audit Plan Management is a planning and data analytics platform that helps you plan projects, allocate resource and manage staff for financial auditing projects. It also helps users explore, analyze and share real-time project analytics data easily.

Filters lets you filter projects based on your parameters in one place, so you can get what you are exactly looking for and find desired information faster with more peace of mind along the way.

Role and Responsibility

Started from research, conceptualising the flows to creating the visual design, conducting user feedback sessions and communicating the interaction to developers.

Problem

Earlier, filters in APM were present as a sidebar on the main screen, you must set filter parameter one by one and click on view projects which would display filtered projects. This didn’t seem to solve users problems

1. Setting all the parameters was very cumbersome, and it took the user minimum of 8 clicks to see one filtered project view.
2. The platform didn’t let one save their filter parameters, to apply the frequently used filter again.

We needed a fresh approach to solve the architecture problem of filters and solving user needs at workplace. To get a better idea of what the users really wanted, we did a simple thing: we talked to the users.



User Interviews

Users are profiled based on their department and roles. We derived the key personas and interviewed them to understand their behaviour, problems and motivations.

  1. Auditors who plan the project, audit it, manage and track the resources.
  2. Financial analysts who work with auditors to analyse the issues in the project.
  3. Project owners who track the analytics for a particular project to understand problems and get insights.

User Stories

After understanding their actions, feelings, pain points and desired outcomes we went ahead to write the job stories.

1. As an auditor, I use the filter feature a lot, so I want it to be at a single click of a button.
2. As a financial analyst, I use the same set of parameters for filters very frequently, I would like to save these parameters, so everytime I dont have to set the same set of parameters again and again.
3. As a project owner, I want to see the projects getting filtered dynamically based on the filters I choose. It wastes a lot of time to enter all the parameters and then wait for the results.

Key Insights

The insights gathered from the user interviews were that users felt tiring to enter all the filter parameters now and then, and then further wait for results till everything was entered. They also felt irritated when entering the same filters again, as they used this feature very frequently.

The filter options are being set one by one and there is no real time response after one filter is set.

Goal

Insights helped us define our goal “Design quick & seamless way to set filters in dashboard enabling users to see filtered projects dynamically, save regularly used filter parameters to create their own filters, change architecture to make it more user-friendly and intuitive.”

Proposed Models

After several brainstorming sessions, throwing ideas around and quick paper sketches we came up with several approaches to visualise project information effectively. Keeping in mind the tech constraints, user pain points and the established mental models we proposed two different approaches.

Add filtering functionality in the column name of the dashboard in the main page Add filter option in the form of a button
Pros Very intuitive, the project grid will dynamically show the filtered projects once any column is changed.
Clean, No distractions.		 Less real time rendering, technically more feasible
Cons It might distract the user to see continuously changing grid. Also, more calls to the backend. The user has to follow more steps to reach the same stage.

User Testing

To validate the use cases and the improvements, we conducted a round of user testing with a dozen of participants on a fully functional prototype. We recorded their responses and compared the data.

It was observed that the button was not very intuitive and again did not lead the dynamic filtering of data. So, we went ahead with implementation of adding filtering functionality in the column name of the dashboard in the main page. We also, added a new filter for year on the top, since it was mandatory to view any of the project.

Visual Design

Based on the insights gained & peer reviews, UI design was formed.

Learning & Impact

It was a great learning opportunity for me, where I understood the importance of hierarchy and Information architecture in depth.

Based on user testing performed for the final feedback, we received great response. The time taken now to view filtered projects was dropped by more than 50%.
